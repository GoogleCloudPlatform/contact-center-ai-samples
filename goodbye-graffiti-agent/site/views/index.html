Goodbye Graffiti

Overview

“Goodbye Graffiti” is a demo built on Dialogflow CX inspired by a number of online forms created by city councils and local governments to report graffiti vandalism.

Sometimes we feel more confortable speaking, others typing! The graffiti demo uses the latest Call Companion UI which provides a multi-modal (voice and visual) customer experience throughout your entire conversation with the agent.

Tell Mel you need to report a graffiti vandalism: "Found some graffiti on the fence of my property this morning", "The north-east fence of St George's park has been painted by vandals", etc...

Features

Generators

Generators use Google's latest generative large language models (LLMs), and prompts that you provide, to generate agent responses at runtime.

The demo leverages generators to greet the user and provide a summary of the report

  • Documentation
  • Codelab

Generative fallback

Gen fallback uses Google's latest generative large language models (LLMs) to generate virtual agent responses when end-user input does not match an intent or parameter for form filling.

For example, if the user cannot provide the exact graffit location (which results in a no-match), the agent escalates to a human agent.

  • Documentation
  • Codelab

Prebuilt components

Prebuilt components are a collection of prebuilt flows provided by Dialogflow for common tasks and scenarios.

The address collection prebuilt component assists the user with a step-by-step address collection process in case the request to the Geocode APIs returns zero results or the returned address is incorrect.

  • Documentation

Google Maps Platform

The demo users the Geocoding API to convert an address or a place into latitude and longitude coordinates or to convert the current user's location into an address.

Static Maps are used to to visualize the location of the graffiti on a map
